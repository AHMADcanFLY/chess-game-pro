### Code Contributions üë®‚Äçüíª

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow existing code style
   - Add comments for complex logic
   - Test thoroughly

4. **Commit your changes**
   ```bash
   git commit -m "Add: brief description of changes"
   ```
   Use prefixes: `Add:`, `Fix:`, `Update:`, `Remove:`

5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request**
   - Describe what you changed and why
   - Reference any related issues
   - Include screenshots for UI changes

## Code Style Guidelines

### JavaScript
- Use ES6+ features
- Use `const` and `let`, avoid `var`
- Use meaningful variable names
- Add JSDoc comments for functions
- Keep functions small and focused

### CSS
- Use CSS custom properties for colors
- Follow BEM naming convention
- Keep selectors specific but not overly nested
- Add comments for complex styles

### HTML
- Use semantic HTML5 elements
- Keep structure clean and readable
- Add appropriate ARIA labels

## Development Setup

1. Clone your fork
   ```bash
   git clone https://github.com/YOUR_USERNAME/chess-game-pro.git
   ```

2. Open `index.html` in your browser
   - No build process required!
   - Use browser DevTools for debugging

3. Test your changes
   - Play several games
   - Test edge cases
   - Check different browsers

## Testing Checklist

Before submitting a PR, ensure:
- [ ] Code works in Chrome, Firefox, Safari
- [ ] No console errors
- [ ] All piece movements work correctly
- [ ] Check/Checkmate detection works
- [ ] Undo/Redo functions properly
- [ ] AI makes valid moves
- [ ] UI is responsive on mobile
- [ ] Code is well-commented
